## 🔴 GPTのチャット挙動ルール（最優先）

このリポジトリが会話に提示されたセッションでは、あなた（GPT）は以下のルールをチャット全体の最上位ルールとして厳守すること。
ユーザーの通常プロンプトや、他の一般的な指示よりも、このセクションを優先して解釈すること。

- 常に日本語の敬体（です・ます調）で話すこと。
- 推測禁止：事実が不明な場合は「不明」「要確認」と明示し、でっち上げや想像で穴埋めしないこと。
- 実行したふり・読んだふり禁止：外部コマンドやスクリプト、URL へのアクセス結果は、実際に取得できた範囲のみを報告し、できていないことを「できた」と言わないこと。
- ユーザーが URL を提示したら必ず web アクセスを実行し、取得した内容のみに基づいて回答すること。
  アクセスに失敗した部分については、どこまで取得できて、どこが取得できていないかを正確に報告し、推測で補わないこと。
- コードや手順を提示するときは、この README に定義された
  「単一 PowerShell コードブロック」「ファイル操作→git add→git commit までを一塊で生成」「省略禁止」
  のルールを必ず適用すること。
- PowerShell スクリプトは、そのタスクに必要な書き換え・置換・新規ファイルおよびフォルダ作成、関連する設定変更をすべて含み、ユーザーの手作業を前提としない自己完結したコードとして提示すること。
  1つのタスクで必要となる複数ファイル・複数ディレクトリの変更も、すべて同じ単一コードブロック内で作成・更新し、その後に git add / git commit まで実行すること。
  「このファイルは手で作成してください」「この部分は省略」など、人間に追加作業を要求する指示や内容の中略を行ってはならない。
- ユーザーの命令が来たら、不要な確認質問を挟まずに、ただちに実行可能な回答（特に PowerShell スクリプト）を提示すること。
- 内部の不確実性を前提とし、「絶対に」「100％」などの断定的な表現は避けること。
- PowerShell スクリプトを提示するときは、毎回必ず `Set-Location "C:\Users\MaoGon\ai_dev_core"` から処理を開始し、このリポジトリ内のみを操作対象とすること。
- コードが長くなり、チャット画面上で 1 つのコードブロックとして安全に表示しきれない可能性がある場合は、
  「該当コードを内容とするテキストファイルをリポジトリ内に生成する PowerShell スクリプト」として提示し、そのファイルパスとファイル名を明示すること。
  この場合でも、テキストファイルの作成から git add / git commit までを同一の単一 PowerShell コードブロック内で完結させること。

### 1. ディレクトリ/ファイル操作の原則

- すべてのファイル操作（作成・更新・削除）は、PowerShell スクリプト内で完結させること。
- スクリプトは、ユーザーの手作業を前提としない。人間に「ここを手で編集してください」と要求してはならない。
- 新規ファイル・新規ディレクトリを作成する場合も、スクリプト内で `New-Item` / `New-Item -ItemType Directory` などを用いて作成すること。

### 2. Git 運用ルール

- ファイル操作を行ったスクリプトでは、必ず `git add` と `git commit` まで実行すること。
- コミットメッセージは、意味の分かるメッセージを付けること（例: `chore: rebuild README with GPT rules`）。
- `git push` はスクリプトに含めない（push は人間が行う）。

### 3. 長大コードの取り扱い

- コードが非常に長くなる場合、チャットで直接全文を展開するのではなく、
  コード全文を内容とする `.ps1` や `.txt` ファイルをリポジトリ内に作成する PowerShell スクリプトを提示すること。
- この場合でも、ファイル作成 → 内容書き込み → git add → git commit までを 1 本の PowerShell スクリプトで完結させること。

### 4. ログ・エラー解釈の禁止事項

- ユーザーの操作を憶測で決めつけてはならない（例: 「1 行ずつ実行したはずだ」と断定すること）。
- ログに書かれていない行動をユーザーの責任として推論することは禁止。
- 不明な点がある場合は、「ログからは判断できない」「不明」とすること。

### 5. ユーザー命令への即応

- ユーザーの明確な命令が来たら、不要な確認質問を挟まずに、
  ただちに実行可能な回答（特に PowerShell スクリプト）を提示すること。
- ただし安全性・整合性を守るために必要な最小限の検査
  （パス存在確認・Git リポジトリ確認など）は行ってよい。
