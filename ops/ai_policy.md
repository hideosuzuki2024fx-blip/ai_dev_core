# AI_DEV_CORE — AI / GPT Policy (Official)

本プロジェクトで GPT が作業する際の **絶対遵守ポリシー**。

---

## 1. 技術ルール（コード生成）

- ファイル操作は **単一 PowerShell スクリプト** で完結
- コードは “そのままコピペで動く完全形”
- 省略記号禁止
- ダミーコード禁止
- ディレクトリは正確（誤パス禁止）

---

## 2. Git 運用ルール（git_workflow.md 統合版）

GPT はコード生成のたびに必ず：

1. ファイル作成・更新
2. git add（対象ファイルのみ）
3. git commit（Conventional Commits）

これらを **単一 PowerShell スクリプト** で実行すること。

push は絶対禁止（人間が日末に行う）。

---

## 3. 倫理・動作ルール

- 捏造禁止
- 憶測禁止
- 「読んだふり」禁止
- 「実行したふり」禁止
- 事実ベースのみで応答
- アップロードされたファイルは必ず精読

---

## 4. 応答トーン／言語仕様

- 敬体のみ（タメ口禁止）
- 過度なメタ説明禁止
- 冗長説明禁止、実装優先
- プロジェクト構造（src/, ops/, packages/）に従う

---

## 5. 出力形式ルール

- **単一 PowerShell スクリプト**で出力
- 複数ファイルを作る場合も単一スクリプトで完結
- コメント最小限
- ファイル全文ブロックで出す（省略禁止）
- JSON/Markdown の省略禁止
