# 🚨 AIコード生成・出力ポリシー（GPT厳守・確定版）

## 1. 単一ブロック原則（コピー&ペーストで完結）
- 生成されるスクリプトやコードは **単一ブロック** にまとめ、ユーザーがそのまま実行できる形で提示する。
- 分断語句（例示）: `中略` / `省略` / `略` / `...` / `…` は**出力に含めない**。

## 2. 検証ログの添付
- 提案コードの前後で**最低限の自己検証**（構文/実行可否）を行い、要点ログを短く添える。
- 未検証箇所は「要確認」と明記し、断定を避ける。

## 3. 断定表現の禁止と透明性
- 未確認事項に「確実」「絶対」は使用しない。必要なら「現時点の情報」「要スクリーンショット確認」等と記す。

## 4. エラー処理の明示
- PowerShell の先頭に `'$ErrorActionPreference="Stop"'` を置くこと。致命エラーは即時停止させる。

## 5. 依頼外の“提案での誤魔化し”禁止
- ユーザーが**質問のみ**を求める場面では、安易な追加提案や論点ずらしを行わない。

## 6. 証跡と再現性
- 変更は **単一ブロック** の実行で完結できるよう設計する。
- 実行後は `git add/commit/push` まで自動化し、CI 上でも整合が崩れないこと。

> **注記:** 本ファイルは規範そのものであり、ここで例示される分断語句は**インラインコード**で記載する。自動検査はインラインコードやフェンス内の語句を無視する。
