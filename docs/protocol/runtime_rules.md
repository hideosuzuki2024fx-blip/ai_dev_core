description: >
  GPT が新しいスレッド開始時に必ず実行する「標準起動ルーチン」。
  記憶リセット・構造誤読・前提崩壊を防止する。
---

# ✅ 1. カレントディレクトリ確認
最初に必ず尋ねる：
```
現在のカレントディレクトリを貼ってください。
```
一致すべき値：
```
C:\Users\MaoGon\Projects\ai_dev_core
```
異なる場合：
```
カレントが一致しません。作業不可。修正後に再提示ください。
```

---

# ✅ 2. プロジェクトツリー要求
提示して実行してもらう：
```
Get-ChildItem -Recurse -Name > project_tree.txt
内容を貼ってください。
```
ユーザーが貼ったツリーを GPT は **必ず** 解析する。

---

# ✅ 3. 作業目的確認
必ず聞く：
```
今回の目的を1行で教えてください。
```
目的不明で作業進行は禁止。

---

# ✅ 4. Step-by-step 限定進行
- 1ステップだけ提示
- ユーザーが「次」「完了」と返すまで停止

---

# ✅ 5. 会話重症化時の提案
会話が重くなった場合：
```
会話が重くなっています。snapshot を保存して、新しいスレッドで安全再開できます。
```

---

# ✅ 6. 禁止事項
- 作業の自動継続
- 記憶前提の回答
- カレント推測
- 構造推測
- ファイル生成の勝手判断
- “前回の続きですね？”

---

# ✅ 7. 出力テンプレート（必須）
全回答は次形式：
```
✅ 結論
…要点…

📌 次のアクション（1行）
…PowerShell コマンドなど…
```
