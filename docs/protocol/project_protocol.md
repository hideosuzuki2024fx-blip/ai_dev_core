# 1. カレントディレクトリ固定ルール
GPT は作業開始前に必ずユーザーへ次を確認する：

```
現在のカレントディレクトリ:
C:\Users\MaoGon\Projects\ai_dev_core
```

一致しない場合は **作業中止**。ユーザーへ修正入力を求める。

---

# 2. プロジェクト構造の取得
GPT は次のいずれかの方法で、**推測なしで正確に構造把握**を行う：
- ユーザーが貼り付けるツリー出力
- またはユーザーが実行する：
```
Get-ChildItem -Recurse -Name > project_tree.txt
```
GPT は貼り付けられた内容を必ず読み取り、誤推測を行わない。

---

# 3. プロジェクト内の重要フォルダ定義（推測禁止）
GPT は下記を「存在前提の固定構造」として扱う：
```
src/backend/main.py
src/backend/static/index.html
src/backend/templates/photobook.html
scripts/*.ps1
handoff/*.md
outputs/
```
存在しない場合は推測補完禁止。ユーザーへ事実確認を依頼する。

---

# 4. 目的の明示義務
GPT は毎回次を要求する：
```
今回の作業目的を 1 行で教えてください。
（例：CSV→PDF Photobook 検証、UI 修正、WeasyPrint 調整、Git 整備）
```
目的不明のまま作業継続は禁止。

---

# 5. 禁止事項
- ファイル／パスの推測
- カレントディレクトリの推測
- プロジェクト構造の推測
- 作業の自動継続
- “以前の続きでしょうか？” の発言
- 存在しないコードの前提での回答

---

# 6. 作業手順（Step-by-step）
1. カレント確認
2. プロジェクトツリー受領
3. 目的の確認
4. 作業開始（必ず Step-by-step）
5. ユーザーが「次」または「完了」と返すまで待機

---

# 7. 出力形式テンプレート（GPT 必須）
GPT の全出力は次形式で統一：
```
✅ 結論
…要点のみ…

📌 次のアクション（1行）
…PowerShell or コマンド…
```

---

# 8. セッション重症化時の対応
会話が重くなった場合、GPT は自動で次を提示：
```
会話が重くなっています。snapshot を保存して新スレッドへ安全移行できます。
```
