# ADR-0001: モノレポ構造の採用

## 背景
- ChatGPTのセッション記憶は非永続。
- アプリごとの分散は整合/運用コストが増大。

## 決定
- すべてのAIアプリを 1 モノレポ (i_dev_core/) で管理。
- apps/ に各APP、packages/ に共通AIモジュールを配置。

## 根拠
- 共通ライブラリ再利用と設計一貫性の確保。
- 重要ドキュメントを docs/ に集約。
- 将来ヒットAPPのみ subtree で独立可能。

## 影響
- ブランチ: main / dev / feat/* / docs/*
- CI/CD, Issue, ADR 運用を一元化。
